ADR 010 · Invites

Public preview image toggle.

A per-conversation toggle decides whether an invite carries a visible group photo. The encrypted member-image path runs independently in either state — turning the toggle off only removes the unencrypted URL from future invites.

Accepted · Last reviewed 2026-05-21
01 Two paths 02 Row fields 03 Default state 04 Upload flow 05 Invite payload 06 Tradeoffs 07 Code map
01

Two paths

Member vs. invite preview

Members always see the encrypted group image — that's the path documented in ADR 009. The toggle controls a second, parallel upload: an unencrypted JPEG whose URL is included in the invite payload so people seeing the link before they join get a real preview card.

Where each image is used
Always Encrypted member image AES-256-GCM ciphertext on S3, key in MLS metadata. Visible only to members. Independent of the toggle.
Toggle-gated Public preview URL Unencrypted JPEG on S3. URL is stamped into the invite payload so non-members can see a preview card.
02

Row fields

DBConversation
FieldTypeMeaning
includeInfoInPublicPreviewBoolThe toggle. When true, the next image update uploads both encrypted and public assets.
publicImageURLStringString?The unencrypted URL stamped at upload time. Set to nil when the toggle flips off.
03

Default state

What new conversations start with

New conversations are created with includeInfoInPublicPreview = true (see ConversationWriter.swift). The user can disable the toggle from the conversation's settings; doing so clears publicImageURLString for future invites.

The database column itself defaults to false at the migration level — that default only applies to rows created without going through the writer (currently: mocks). Every conversation created through the normal write path opts into public previews until the user turns the toggle off.

04

Upload flow

ConversationMetadataWriter.updateImage()
01 Writer Encrypt the JPEG with AES-256-GCM under the group's imageEncryptionKey; upload ciphertext to S3.This step always runs, regardless of toggle state.
02 Writer Read localConversation.includeInfoInPublicPreview.If false, stop here. publicImageURLString remains nil.
03 Writer If true, generatePublicPreviewUrl() uploads an unencrypted JPEG to S3 and stamps the URL into publicImageURLString.
04 Toggle off later updateIncludeInfoInPublicPreview(false) sets the bool to false and nils publicImageURLString.The encrypted member image is unaffected; existing S3 objects age out per ADR 008.
05

Invite payload

SignedInvite+Signing

When the slug is generated, the invite payload's imageURL field is populated from publicImageURLString — never from any encrypted ref. A conversation with the toggle off ships an invite payload that has no image URL; the receiver renders the deterministic emoji from ADR 007 instead.
