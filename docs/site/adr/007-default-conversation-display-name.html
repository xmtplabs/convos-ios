ADR 007 · Conversations

Default display name and emoji.

Every conversation needs a name and an avatar — even when no-one has set them. The algorithm below is the cross-client contract: same conversation, same emoji, same fallback name on every device that follows it.

01

Display name resolution

First matching row wins

"Other members" means every member excluding the current user. A member's displayName is their profile name if set, otherwise "Somebody".

PriorityConditionDisplay name
1Conversation has a non-empty custom nameThe custom name.
2DM, other member has a nameOther member's displayName.
3DM, other member has no name"Somebody"
4Group with no other members"New Convo"
5Group with other membersResult of member name formatting (next section).
02

Member name formatting

Up to MAX_NAMES = 3

Partition other members into named (profile name is non-empty, sorted lexicographically) and anonymous (counted, never listed individually). Then apply one of three cases.

Case A — no named members

Anonymous countOutput
0"" (empty)
1"Somebody"
2+"Somebodies"

Case B — total ≤ 3 items

Combine the sorted named profiles with one "Somebody" / "Somebodies" token (when anonymous > 0), then join.

Item countSeparatorExample
1—"Alice"
2 & "Alice & Bob"
3, "Alice, Bob, Somebody"

Case C — total > 3 items

Take the first 3 sorted named profiles; the rest collapse into an "and N other(s)" tail. Anonymous members are counted toward the tail, never listed individually.

Members (names)Output
["Alice"]"Alice"
["Bob", "Alice"]"Alice & Bob"
["Carol", "Alice", "Bob"]"Alice, Bob, Carol"
["Dave", "Carol", "Alice", "Bob"]"Alice, Bob, Carol and 1 other"
["Eve", "Dave", "Carol", "Alice", "Bob"]"Alice, Bob, Carol and 2 others"
[nil]"Somebody"
[nil, nil]"Somebodies"
["Alice", nil]"Alice & Somebody"
["Alice", nil, nil]"Alice & Somebodies"
["Alice", "Bob", nil]"Alice, Bob, Somebody"
["Alice", "Bob", "Carol", nil]"Alice, Bob, Carol and 1 other"
03

Emoji selection

Deterministic from the group ID

Collisions are expected — 90 emojis across many conversations means two unrelated conversations can land on the same icon. That's fine; the emoji is one of several visual cues, not a unique identifier.

Avatar resolution

Custom · single peer · emoji · cluster
PriorityConditionAvatar
1Conversation has a custom imageURLThe custom image.
2Exactly 1 other memberThat member's profile avatar.
30 other members, or no member has an avatarThe deterministic emoji.

06

Fully anonymous flag

Signal for presentation tweaks

A conversation is "fully anonymous" when it has at least 1 other member and none of those members have a non-empty profile name. A conversation with 0 other members is not anonymous — it's considered new. Clients may use the flag to lean into the emoji presentation when there's no name to show.
