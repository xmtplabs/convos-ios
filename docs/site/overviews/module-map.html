03

Library products

Three products from one package

The ConvosCore SPM package vends three library products. Splitting them at the product level (not at the package level) means a single dependency graph but selectable surfaces — the NSE pulls only what it needs; tests run against the cross-platform core without iOS.

Products from the ConvosCore package
ConvosCore Cross-platform business logic Compiles on iOS and macOS — never imports UIKit. Uses the ImageType alias for cross-platform image handling.
ConvosCoreiOS iOS-only bridges Implements ConvosCore protocols with iOS-specific APIs: UIImage handling, push registration, system notification helpers.
ConvosConnectionsXMTP XMTP wire layer for device connections Three JSON codecs sitting between ConvosConnections Swift types and XMTPiOS. Lets the device package stay free of any XMTP dependency.

App targets

What ships to users
TargetRolePulls in
Convos The main app. SwiftUI views, ViewModels, deep-link handling, settings. ConvosCore + ConvosCoreiOS + ConvosConnectionsXMTP (registered codecs).
ConvosAppClip 10MB-budget invite preview & join surface. Writes identity to the shared keychain so the main app picks it up post-install. See ADR 011 §06. ConvosCore + ConvosCoreiOS via the app-group access group.
NotificationService NSE — decrypts MLS push payloads, validates clientId, surfaces the notification body. ConvosCore (identity-keyed cache via CachedPushNotificationHandler).
ConvosTests Integration suites for the main app target. Convos + ConvosCore.
05

Placement rules

Where new code goes

When you're about to add a new file, this is the lookup table the codebase wants you to use. The rules are present-tense — they describe what holds today, not what would happen ideally.

If the new code…Goes in
…handles a SwiftUI view, ViewModel, or design-system surfaceConvos/ (main app target)
…is business logic, a writer/repository/worker, or a network callConvosCore/Sources/ConvosCore/
…uses UIKit (UIImage, UIApplication, NotificationCenter for iOS pushes)ConvosCore/Sources/ConvosCoreiOS/
…is an iOS-only system integration the core needs (e.g. push registration)Define a protocol in ConvosCore, implement it in ConvosCoreiOS, inject from the main app.
…is a new device-data source or sinkConvosConnections/Sources/ConvosConnections/
…is the XMTP wire format for a device-connections messageConvosCore/Sources/ConvosConnectionsXMTP/
…is invite-token cryptography (no XMTP)ConvosInvites/Sources/ConvosInvitesCore/
…is the app-side glue around an invite (sending, custom metadata)ConvosCore/Sources/ConvosCore/Invites & Custom Metadata/
…is a new XMTP custom content typeConvosCore/Sources/ConvosCore/Custom Content Types/ + register in SessionStateMachine.clientOptions
…is a GRDB migrationConvosCore/Sources/ConvosCore/Storage/SharedDatabaseMigrator.swift
Cross-platform discipline. Code in ConvosCore must compile on macOS — never import UIKit, never #if canImport(UIKit). Use the ImageType alias from ConvosCore/Utilities/ImageType.swift for cross-platform image handling. Anything iOS-specific goes in ConvosCoreiOS behind a protocol injected at the app level.
